Werewolf-AI

面向多智能体协作、博弈与自进化的狼人杀系统

我们构建了一个以对局引擎为推理内核的多智能体狼人杀系统。引擎负责规则状态机与服务端信息隔离， 对真人与 AI 提供同一套接入协议；Agent 作为独立决策服务，基于引擎下发的可见状态完成身份推断、策略召回与行动决策， 并在终局通过复盘把经验沉淀为跨局策略记忆；实验平台负责批量编排对照实验、断点续跑与全链路可观测分析。 在此之上，我们验证了跨局自进化对好人阵营胜率、投票质量与中后期推理能力的显著提升。

1系统构成

三个边界清晰、可独立演进的服务，通过公开协议协作。规则正确性、Agent 决策与实验编排互不耦合。

game-system推理内核

房间与多局生命周期、规则状态机、角色技能调度、public/player/god 三类视角裁剪、pending action 协议、事件流、状态快照与 replay frame。它只回答「这局发生了什么」，不理解模型与策略。

agent玩家决策服务

Session loop 驱动的 Profile-driven Agent。基于引擎下发的可见状态做 section 化上下文拼接，执行 belief → strategy → decision → postgame review，维护 room-scoped 策略记忆，并落库全链路 LLM trace。

experiments批量编排与分析

多房间并行、多局串行、固定角色与公开名重排；断点续跑与自愈；实时观战、细粒度回放（含 LLM call）与多切面离线分析脚本。只编排，不复制局内事实。

2核心设计

几个我们认为值得展开的工程与方法决策。

isolation

服务端信息隔离

三类视角在后端统一裁剪，覆盖事件、WebSocket、快照与回放。Agent 拿到的始终是裁剪后的可见状态，无需也无法自行维护隐藏事实。

homogeneous

控制器无差别接入

真人、AI 与实验 runner 走同一套 join / ready / pending / submit 协议。同一引擎既支撑真人体验，也支撑批量 Agent 实验。

profile

Profile 表达实验变量

baseline、belief、static strategy、self-evolve 等变体由 profile 表达，一套 runtime 服务所有变体。session 创建即固化 config 快照，历史实验可解释。

context

Section 化上下文

每次 LLM call 的输入直接由可见状态组装成带标题、来源、优先级的 section，而非维护长 message 历史。信息边界清晰、可审计。

belief

带证据的身份推断

belief 结合上一轮推断保持一致性，要求输出置信度与 evidence 以抑制幻觉，并完整落库每次推断快照供细粒度机制分析。

self-evolve

受约束的自进化

复盘只能写入 room-scoped 策略记忆，不改代码或固定策略包；helpful 加分由确定性规则完成。下一局按 role/phase/归一化得分召回。

3主要结果

两个自进化对照实验各 8 房间 × 20 局（共 160 局），考察同一房间内连续对局的跨局学习效果。

+27.5pp 好人自进化胜率 S4 − S1（47.5% → 75.0%）
+14.2pp 好人白天投中狼人率提升
+2.5pp 狼人自进化胜率 S4 − S1（几乎无增益）

① 好人自进化有效。胜率、投票命中率、中后期身份推断三条证据链一致上升。

② 狼人自进化收益有限。伪装略有改善，但受胜负机制约束，未能转化为胜率。

③ 增益集中在中后期。记忆复用在信息更丰富的 day≥2 阶段才充分发挥作用，而非开局首日。
