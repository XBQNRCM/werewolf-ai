Agent 认知架构

一套 Profile-driven runtime 服务所有角色与对局变体。Agent 不维护长对话历史，而是基于引擎下发的可见状态 做 section 化上下文拼接；belief、strategy memory 与 postgame review 共同构成可消融、可复盘、可自进化的决策闭环。

1Session Loop 与 Profile

实验变量由 profile 表达，而非复制代码或长期分支。session 创建时即固化完整 config 快照，使历史实验始终可解释。

session创建加载 profile · join · 首次 ready
poll轮询取最新 game 的 pending
belief推断可选 · 按 bucket 触发
strategy召回static pack + room memory
decision决策约束在 legal_actions 内
submit提交POST actions
review复盘终局写 memory · 再 ready

同一 runtime 通过替换 profile_id 就能比较「是否推断」「推断是否进入行动」「是否注入静态打法」「是否允许局后自进化」等变量。profile 还承担模型路由：module_models 允许 belief、decision、postgame review 各用不同模型、温度与超时。

ProfileBelief进入决策StrategyPostgame用途
baseline_v1yesnononeno预测可观测、决策不吃推断的基线
belief_decision_v1yesyesnoneno验证推断进入决策的收益
static_strategy_v1yesyesstaticno验证只读打法知识
self_evolve_v1yesyesroom_memoryyes完整自进化实验组

2Section 化上下文管理

每次 LLM call 的输入直接由引擎传来的可见状态组装，整理成带标题、来源与优先级的 ContextSection 再按优先级渲染，而非维护一段不可审计的长 prompt。

belief prompt

身份推断上下文

  • 【对局玩家信息】私有身份与可见玩家状态
  • 【本局规则】rules_text
  • 【当前可见时间线】裁剪后的事件
  • 【上一轮身份推断摘要】同局链式传递
  • 【推断经验】phase=belief 的 room memory
decision prompt

行动决策上下文

  • 【行动指令】+【合法行动】逐条列出
  • 【对局玩家信息】+【当前可见时间线】
  • 【最近一次身份推断摘要】profile 允许时
  • 【基础打法】static + 【对局经验】memory top 5
  • 【狼队夜聊协商】狼人夜间协作场景

每个 section 都保留「信息来自哪里」：规则来自 game-system，推断来自 agent cache，策略来自 static pack 或 room memory，复盘真相来自 god replay。排查坏 case 时可直接回看 llm_calls.request_payload.messages，确认模型当时看到的信息边界。

3Belief：带证据的身份推断

belief 只做可见视角内的认知建模。它结合上一轮同局推断保持推理一致性，并要求模型输出置信度与 evidence——LSPO 的消融实验表明这类结构化约束能有效抑制幻觉。

字段含义
deduced_role当前最可能身份（受限于角色枚举）
role_confidence对推测身份的自报置信度（1–10）
statement_reliability该玩家发言可信度（0–10）
reasoning / evidence简短推理与引用可见事件的可读证据

触发与缓存

  • 除第一晚外，每次行动前触发 full deduction
  • _belief_bucket 让狼人夜聊后的 kill 复用同一轮推断
  • 模型上下文只保留同局最近一次 deduction
  • 数据库保留每一次 snapshot，供细粒度机制分析
  • 新 game_id 清空 cache，避免跨局污染

失败降级

  • JSON / 枚举 / 越界校验失败 → repair 重试一次
  • repair 仍失败 → deterministic fallback
  • fallback 给已知身份 10 分，其余默认 villager / 5 分
  • 首次错误与 llm_call_ids 写入 validation_result

4Strategy Memory 与自进化

自进化被刻意收窄成 memory 写入：局后复盘可以提出新经验，但只能写入 room_id + agent_identity_id 范围的策略记忆，不改代码、不改固定策略包，也不让模型给自己打分。

召回规则（确定性）

  • 按 role / phase 过滤匹配
  • age-normalized score 排序：score / lifespan
  • 精确 role/phase 优先于通用（null）
  • 取 top 5 注入 decision 或 belief prompt
  • 软约束：当前局面与 legal_actions 优先级最高

Score 更新（确定性）

  • 出现在 helpful_memories：+8
  • 本阵营获胜且 helpful：额外 +2
  • 上限 100；第一版不做失败扣分
  • new_memories 不得写入具体玩家公开名

Postgame Review

复盘 prompt 的核心价值是把「模型当时知道什么」与「终局真相是什么」放在同一条时间线上：它能区分 belief 错误（当时推断错了）与 decision 错误（有信息但行动没用好）。

输入：god replay + 本局 deduction/action 轨迹 + 已有 active memory + 胜负结果
输出：summary · belief_errors · decision_errors · helpful_memories(≤4) · new_memories

自进化闭环：
  多局对战 → 保存 belief/action/replay 轨迹
  → 终局复盘产生少量候选经验 → 写入 room-scoped memory
  → 下局按 role/phase/归一化得分召回 → 决策在软约束下参考经验

每条 memory 都可追溯到 source_game_id、profile_id、postgame_review_id 与当时的 LLM call。这让自进化更像「经验召回」而非把历史结论当作新事实灌入，也保留了版本回溯能力。

5持久化与 LLM Trace

可观测性是一等公民。每次 deduction、decision、postgame review 与真实 LLM 调用都有结构化记录，并通过 ID 互相链接。

表记录内容
agent_sessionssession 生命周期与 resolved config 快照
deduction_runs / deduction_players每次 belief update 及 per-player 判断
action_decisions行动决策、选用的 strategy/memory、validation 结果
strategy_memoriesroom-scoped 自进化策略记忆
postgame_reviews复盘摘要与 belief/decision 错误分析
llm_calls真实请求/响应审计，链接到具体 action / deduction / review

解耦思考与提交：已保存但未成功提交的 decision 会被复用、重新 submit，而不是重新调用 LLM——避免一次网络失败触发重复思考与行为漂移。终局处理完成后再次 set_ready(true)，以支持同房多局串行。
